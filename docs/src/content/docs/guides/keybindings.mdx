:::tip[Same viewer inside the terminal]
This keyboard isn't just docs. Open the help modal (`?`), and on the **Hotkeys** tab click
`[K ⌨ keyboard]` (or press `K`) to get the identical viewer in the TUI — built from the same
keymap, nav cluster on the right. It's responsive: a full-blown bordered keyboard when the
terminal is large, a compact strip when it isn't. Every keypress — including physical
`Shift`/`Ctrl` on terminals with the Kitty keyboard protocol — highlights its key and lists
what it does in a panel below (whose keys column auto-sizes to the longest combo shown); `Esc`
closes it.
:::

## Notes

- **Retry vs refetch** — `r`/`R` only re-run repos that need it (failed or skipped);
  `e`/`E` re-run any repo even if it was already up to date. In the status bar, `r`/`R`
  dim when no repo has an issue, and `e`/`E` dim when nothing is eligible.
- **Sort the list** — press `s` (or click the `s sort ⟪…⟫ ▾` header trigger) to open the
  sort dropdown, then pick with the mouse, the `↑↓` arrows + `Enter`, or a row's mnemonic key —
  name / branch / status / ahead-behind / dirty / last-commit / worktrees / branches / stashes /
  pulled / changed; clicking a column header also sorts. Re-pick or re-click flips `▲`/`▼`. The
  list is always sorted (Name asc by default) and the order persists across runs.
- **Status filter on `f`** — `f` then `a`/`u`/`c`/`s`/`f`/`i` filters the list by
  status (on top of the `/` name filter).
- **Views & folding** — `v g` toggles the [grouped list view](../groups/) and `v t` the
  [directory-tree view](../tree-view/) (independent toggles → flat / grouped / tree / tree+groups);
  `Z` re-resolves dynamic (`command`/`url`) group memberships. On a folder or collapsible group
  header, `Enter`/`Space`/click collapse or expand the section; tree-style, `←` collapses the
  selected header or jumps to its parent while `→` expands. Fold everything with `-` (collapse
  all), `+`/`=` (expand all), `*` (expand subtree), or the vim-style `z` chord
  (`za`/`zo`/`zc`/`zO`/`zM`/`zR`).
- **Throttling** — if a remote rate-limits the run (HTTP 429 / "rate limit" / SSH connection
  throttling), those repos get a distinct **throttled** state (`↯`), a warning banner appears,
  concurrency is automatically halved, and the repos are re-queued with exponential backoff.
- **Panels** — `Tab`/`Shift-Tab` cycle focus across the visible panels (`[1]` list · `[2]` info ·
  `[3]` result · `[4]` repo page), or `1`-`4` jump directly; the focused panel shows a bright
  rounded border. The divider shows a grip and highlights while you drag it.
- **`l` opens [lazygit](https://github.com/jesseduffield/lazygit)** — lazygit is a fast,
  full-featured terminal UI for git (stage hunks, commit, branch, merge, rebase, cherry-pick,
  resolve conflicts, browse the log, and more). `l` suspends `polygit` and opens lazygit in the
  selected repo (or the selected row's path on the repo page), returning you here when you quit it
  (`q`). If lazygit isn't [installed](https://github.com/jesseduffield/lazygit#installation), a
  note appears instead of launching.
- **Checkout moved to `Shift`+`Enter`** — on the repo page, plain `Enter` (and a
  double-click) opens the diff for a stash or dirty row. `Shift`+`Enter` checks out the
  selected clean, non-current branch, so the two actions no longer collide.

  :::caution[Terminal support]
  `Shift`+`Enter` relies on the Kitty keyboard protocol to be reported distinctly.
  `polygit` enables it when the terminal supports it (recent Windows Terminal, WezTerm,
  kitty, foot). On terminals without support, `Shift`+`Enter` falls back to plain `Enter`.
  :::
- **Web-like mouse support** — whole status-bar hints are clickable (not just the key), the list
  pane's top-border triggers `f status ⟪…⟫ ▾ · s sort ⟪…⟫ ▾ · t cols ▾` open the status-filter /
  sort / columns dropdown (the sort trigger carries the
  active `⟪sort ▲⟫`), the active `[filter]`/`{status}` tags sit next to their hints and click to
  clear/reset, every modal has an `[x]` and closes on an outside click, settings rows and
  radio chips are clickable, confirm dialogs have clickable `[y]`/`[n]`, and the repo page has
  an `esc✕` close button (next to its `m▢`/`m▣` maximize button).
- **Clickable commands** — the action hints in the status bar (and the columns/sort dropdowns)
  are mouse-clickable; clicking runs the same command as the key.
- **Scrollbars** — the repo list, the info pane, the preview, the help modal, and the repo
  page all show a scrollbar when their content overflows; click or drag it to scroll (the
  wheel scrolls whichever pane is under the cursor). The list's scrollbar sits right beside
  the pane divider, and a grab on it scrolls the list rather than starting a splitter drag.
